什么是强化学习？

📖 一句话定义

强化学习（Reinforcement Learning, RL）是机器学习的一个分支，研究智能体如何在环境中通过试错来学习最优行为策略，以最大化累积奖励。

核心概念

概念 说明 类比
智能体 Agent 执行动作的决策者 玩游戏的人
环境 Environment 智能体所处的外部世界 游戏规则和场景
状态 State 环境在某一时刻的描述 游戏画面
动作 Action 智能体可以执行的操作 按键操作
奖励 Reward 环境对动作的反馈 得分或惩罚

强化学习 vs 其他机器学习

监督学习
有标签数据，学习输入→输出的映射
例如：图像分类、房价预测
无监督学习
无标签数据，发现数据内在结构
例如：聚类、降维
强化学习
通过试错学习，延迟奖励
例如：游戏AI、机器人控制

Q-Learning 算法

什么是Q值？

Q值（Quality Value）表示在某一状态下执行某一动作的期望累积奖励。

Q(s, a) = 在状态 s 执行动作 a 后，能获得的期望总奖励

核心更新公式

Q(s,a) ← Q(s,a) + α × [r + γ × max(Q(s',a')) - Q(s,a)]

参数说明

符号 名称 取值范围 作用
α 学习率 (Learning Rate) 0 ~ 1 控制新信息覆盖旧信息的程度
γ 折扣因子 (Discount Factor) 0 ~ 1 控制未来奖励的重要性
s 当前状态 - 执行动作前的状态
a 执行的动作 - 在状态s下选择的动作
r 即时奖励 - 执行动作后获得的奖励
s' 下一状态 - 执行动作后的新状态

探索 vs 利用（Exploration vs Exploitation）

强化学习面临一个根本问题：是选择当前已知的最佳动作，还是尝试新动作以发现更好的策略？

⚖️ 探索-利用困境
  • 利用 (Exploitation)：选择当前Q值最高的动作 → 获得即时最大奖励
  • 探索 (Exploration)：随机尝试新动作 → 可能发现更好的策略

ε-贪心策略（Epsilon-Greedy）

P(选择动作a) = ε 时随机选择， 1-ε 时选择最优动作

训练过程中 ε 会逐渐衰减：

  • 训练初期：ε 较大（如 1.0），多探索
  • 训练后期：ε 较小（如 0.01），多利用

环境代码详解

GridWorldEnv 类

grid_world_env.py 
class GridWorldEnv:
    def __init__(self, size=4):
        # 环境初始化
        self.size = size                    # 网格大小 4×4
        self.start_pos = (0, 0)            # 起点
        self.goal_pos = (3, 3)             # 终点
        self.traps = [(1, 1), (2, 2)]   # 陷阱位置
        self.action_space = 4             # 4个动作（上下左右）

核心方法：reset()

重置环境 
def reset(self) -> int:
    """重置环境，返回初始状态"""
    self.current_pos = self.start_pos  # 回到起点
    self.steps = 0                      # 重置步数
    return self._get_state()            # 返回状态索引

核心方法：step(action)

执行动作 
def step(self, action: int) -> tuple:
    """
    执行动作，返回 (新状态, 奖励, 是否结束, 信息)
    动作: 0=上, 1=下, 2=左, 3=右
    """
    row, col = self.current_pos
    
    # 根据动作移动位置（带边界检查）
    if action == 0:    # 上
        row = max(0, row - 1)
    elif action == 1:  # 下
        row = min(self.size - 1, row + 1)
    elif action == 2:  # 左
        col = max(0, col - 1)
    elif action == 3:  # 右
        col = min(self.size - 1, col + 1)
    
    self.current_pos = (row, col)
    
    # 计算奖励
    reward = -1  # 每步基础惩罚
    if self.current_pos == self.goal_pos:
        reward = 10   # 到达终点
        done = True
    elif self.current_pos in self.traps:
        reward = -10  # 踩中陷阱
        done = True
    else:
        done = False
    
    return self._get_state(), reward, done, {}
📝 状态编码

环境将二维坐标 (row, col) 编码为一维状态索引：state = row × 4 + col

例如：(0,0)→0, (0,1)→1, (3,3)→15

智能体代码详解

QLearningAgent 类

q_learning_agent.py - 初始化 
class QLearningAgent:
    def __init__(self, n_states, n_actions, 
                 learning_rate=0.1, 
                 gamma=0.99,
                 epsilon=1.0,
                 epsilon_decay=0.995,
                 epsilon_min=0.01):
        
        # Q表: 每个状态-动作对的初始Q值为0
        self.q_table = np.zeros((n_states, n_actions))
        
        self.lr = learning_rate          # 学习率 α
        self.gamma = gamma               # 折扣因子 γ
        self.epsilon = epsilon           # 探索率 ε
        self.epsilon_decay = epsilon_decay    # 衰减率
        self.epsilon_min = epsilon_min   # 最小探索率

动作选择：ε-贪心策略

选择动作 
def choose_action(self, state: int, training=True) -> int:
    """
    ε-贪心策略:
    - 以 ε 概率随机选择（探索）
    - 以 1-ε 概率选择Q值最大的动作（利用）
    """
    if training and np.random.random() < self.epsilon:
        return np.random.randint(self.n_actions)  # 探索
    else:
        q_values = self.q_table[state]
        return np.argmax(q_values)  # 利用

核心：Q值更新

学习（更新Q值） 
def learn(self, state, action, reward, next_state, done):
    """
    Q-Learning更新规则:
    Q(s,a) = Q(s,a) + α * [r + γ * max(Q(s',a')) - Q(s,a)]
    """
    # 当前Q值
    current_q = self.q_table[state, action]
    
    # 计算TD目标值
    if done:
        td_target = reward  # 结束状态没有未来奖励
    else:
        # 下一状态的最大Q值 × 折扣因子 + 即时奖励
        td_target = reward + self.gamma * np.max(self.q_table[next_state])
    
    # TD误差 = 目标值 - 当前值
    td_error = td_target - current_q
    
    # 更新Q值（向目标值移动一小步）
    self.q_table[state, action] += self.lr * td_error
✅ 代码要点
  1. TD(0)更新：使用当前估计来更新当前估计（自举）
  2. 离线学习：更新时使用的是max(Q(s'))，不依赖实际采取的下一个动作
  3. 增量更新：学习率控制更新幅度，避免震荡

训练流程详解

train.py - 训练循环 
def train(n_episodes=1000):
    env = GridWorldEnv(size=4)     # 创建环境
    agent = QLearningAgent(...)      # 创建智能体
    
    for episode in range(n_episodes):
        state = env.reset()          # 1. 重置环境
        total_reward = 0
        done = False
        
        while not done:
            # 2. 选择动作
            action = agent.choose_action(state)
            
            # 3. 执行动作，观察新状态和奖励
            next_state, reward, done, _ = env.step(action)
            
            # 4. 学习（更新Q值）
            agent.learn(state, action, reward, next_state, done)
            
            # 5. 更新状态
            state = next_state
            total_reward += reward
        
        # 6. 衰减探索率
        agent.update_epsilon()

结果分析

训练曲线解读

📊 奖励曲线

随着训练进行，每轮的总奖励应该呈现上升趋势：

  • 初期：奖励波动大，可能经常踩中陷阱
  • 中期：奖励逐渐提高，开始找到路径
  • 后期：奖励稳定，形成最优策略

Q表分析

训练完成后，Q表应该呈现出以下特征：

位置 预期Q值特征 原因
靠近终点 较高（正向） 距离奖励近
靠近陷阱 较低（负向） 避免惩罚
起点 中等（路径相关） 最优路径的起始
⚠️ 常见问题
  • Q值不收敛：尝试减小学习率或增加训练轮数
  • 总是踩陷阱：检查奖励设置是否合理
  • 步数过多：增加时间惩罚或调整折扣因子

参数调优指南

关键参数

参数 默认值 调优建议
learning_rate 0.1 太小：学习慢
太大：不稳定震荡
建议：0.01 ~ 0.3

  2. 一次只改一个参数：便于观察效果
  3. 观察训练曲线：判断是否收敛
  4. 网格搜索：系统地尝试参数组合

进阶：DQN算法

从Q表到神经网络

当状态空间非常大或连续时（如游戏画面），无法使用表格存储Q值。深度Q网络（DQN）使用神经网络来近似Q函数。

🧠 DQN核心改进
  1. 神经网络近似：处理连续/大规模状态空间
  2. 经验回放：存储经验，随机采样训练
  3. 目标网络：稳定训练过程

网络结构

DQN网络结构 
输入层:  state_size (状态维度，如CartPole的4维)
   ↓
隐藏层1: 64神经元, ReLU激活
   ↓
隐藏层2: 64神经元, ReLU激活
   ↓
输出层:  action_size (每个动作的Q值，如CartPole的2维)

详见 cartpole_dqn.py 文件，包含完整的DQN实现。
